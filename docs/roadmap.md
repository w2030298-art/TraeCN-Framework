# 路线图

## v1.4.0 目标

`v1.4.0` 聚焦于把当前框架从“已具备治理骨架”推进到“可持续协作与可持续演进”的阶段。

## 重点方向

### 1. 协作入口完善

- 补齐 `Issue` 模板体系，统一缺陷与需求收集方式
- 建立路线图文档，明确版本目标、优先级和边界
- 让文档、Issue、PR、Release 形成统一入口链路

### 2. 文档与治理闭环

- 持续收敛公开文档入口，减少重复说明
- 明确哪些内容属于公开仓库，哪些属于本地运行态
- 让版本更新、发版说明与协作规范保持一致

### 3. 框架演进准备

- 梳理可进入 `Modules/` 的复用资产
- 评估是否需要增加更多内建模板或自动化脚手架
- 为后续更高阶能力预留清晰的版本边界

## 非目标

- 不在 `v1.4.0` 内引入大规模破坏性重构
- 不在治理文档尚未稳定前扩散过多实验性模块
- 不把本地运行态资产重新暴露到公开仓库根目录

## 优先级

1. 先完成治理闭环文档和协作入口
2. 再整理复用资产与自动化模板
3. 最后评估更高阶能力的引入时机

## 进入标准

当以下条件满足时，可认为 `v1.4.0` 达到可发布状态：

- 协作入口完整，`Issue`、`PR`、`Release` 模板可直接使用
- 文档导航覆盖公开入口、版本说明、路线图和发版流程
- 治理相关改动可以通过版本记录追踪
