# TraeCN-Framework

> 专为 Trae IDE 打造的 AI 编程助手框架。基于本地结构化记忆、Coordinator 编排和 Skill 技能调度。

[![Version](https://img.shields.io/badge/version-1.4.2-blue.svg)](https://github.com/w2030298-art/TraeCN-Framework)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

## 这是什么

这是一个面向 `Trae IDE` 的 AI 编程助手框架，用结构化记忆、Coordinator 编排和 Skill 调度，把“会话式助手”升级成“可持续协作的开发伙伴”。

## 当前状态

- **最新正式版**：`v1.4.2`
- **项目阶段**：核心闭环已补齐，进入可公开使用与维护的稳定阶段
- **主要更新**: 引入三种便捷初始化方式（快捷自动 / 链接导入 / 本地索引），支持在任意新工作区一键部署 `.trae/` 框架。

## 为什么值得看

- 它不只是提示词集合，而是把规则、技能、记忆、生命周期和发布治理做成了一个完整框架
- 它默认强调“真实执行、自动诊断、结果验证、经验沉淀”，而不是只追求触发词和模板数量
- 它可以作为你自己的 Trae 工作基座，也可以作为一个可继续衍生的开源起点

## 你会得到什么

- **结构化记忆**：把用户偏好、项目状态、会话交接和知识沉淀拆成可维护文件
- **Skill 调度**：把分析、设计、测试、部署等专业任务从主会话中隔离出来
- **自愈闭环**：以“修改 -> 诊断 -> 修复 -> 验证”为默认工作流
- **流程产品化**：`/初始化`、`/重置`、`/结束` 与核心 Skill 都有前置检查、验收标准、失败处理和沉淀出口
- **公开治理**：同时具备版本管理、贡献规范、Issue/PR/Release 模板和路线图

## 核心理念

传统方式用纯文本文件和庞大 Prompt 维护记忆和角色，容易出现记忆污染、多角色冲突和维护成本上升的问题。

本框架的目标是把这些能力拆成稳定的结构层：

- **Coordinator 模式**：主 Agent 负责任务拆解、生命周期管理和调度
- **开放衍生**：框架是持续演进的，而不是一次性 Prompt
- **感知先行 (Cascade Mode)**：实时监控 Linter、终端输出和隐式意图，实现“问题未报，修复先行”

## 分层架构

框架采用五层结构：

| 层级 | 职责 |
|------|------|
| Coordinator | 解析用户意图、维护 Todo、调度 Skill 或执行内建流程 |
| Rules | 定义身份、记忆、调度、错误处理、自动保存和输出规范 |
| Skills | 承担分析、设计、测试、部署、复盘、归档等专业任务 |
| Memory | 保存用户偏好、项目状态、会话交接和知识沉淀 |
| Workspace | 承载归档、自定义技能、进化记录和备份 |

## 仓库入口约束

- 根目录只保留公开入口文件与一级导航，例如 `README.md`、`CHANGELOG.md`、`docs/`、`.trae/`

## 快速开始

### 15 秒上手（推荐）

1. **一次性全局安装**（每台机器只需执行一次）：

```powershell
# Windows
irm https://raw.githubusercontent.com/w2030298-art/TraeCN-Framework/main/scripts/install.ps1 | iex

# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/w2030298-art/TraeCN-Framework/main/scripts/install.sh | bash
```

2. **在任意新工作区输入**：

```text
/初始化
```

AI 助手将自动部署 `.trae/` 框架、创建目录结构、初始化记忆文件，然后直接开始对话协作。

### 三种初始化方式

| 方式 | 触发格式 | 适用场景 |
|------|----------|----------|
| **快捷自动** | `/初始化` | 最常用。本地有缓存则直接部署，无缓存则自动从 GitHub 下载 |
| **链接导入** | `/初始化 <url>` | 使用自定义配置源或特定版本（如 fork 的仓库） |
| **本地索引** | `/初始化 @<索引>` | 使用本地预存的配置模板，完全离线可用 |

### 传统方式（手动复制）

如果你不想使用全局安装，也可以手动将 `.trae/` 文件夹复制到项目根目录，然后输入 `/初始化`。

### 详细流程

#### 1. 安装全局初始化 Skill（一次性）

运行安装脚本，将 `traecn-init` Skill 安装到 `$HOME/.trae-cn/skills/`，同时在 `$HOME/.trae-configs/` 创建默认配置模板：

```powershell
# Windows（已克隆仓库）
.\scripts\install.ps1 -Local

# macOS / Linux（已克隆仓库）
./scripts/install.sh --local
```

安装完成后，`/初始化` 将在**任意工作区**可用。

#### 2. 初始化工作区

在 Trae IDE 中打开工作区，在聊天框输入：

```text
/初始化
```

AI 助手将自动完成：

- 检测 `.trae/` 是否存在（不存在则自动部署）
- 创建 `Core/记忆/` 等目录结构
- 生成空白记忆文件
- 配置用户偏好
- 检查是否存在可恢复的旧会话

如需从特定源初始化：
- `/初始化 https://github.com/xxx/xxx/archive/main.zip` — 链接导入
- `/初始化 @default` — 本地索引（离线）

#### 3. 开始协作

完成初始化后，你可以直接描述任务，或使用内建命令和 Skill 指令进入不同工作流。

#### 4. 常用指令

框架将指令分成两类：

- **Coordinator 内建指令**：由主 Agent 直接执行
- **Skill 指令**：由主 Agent 调度到对应技能

| 指令 | 类型 | 功能 |
|------|------|------|
| `/初始化` | 内建指令 | 新用户首次使用，创建目录和记忆文件 |
| `/重置` | 内建指令 | 清除用户数据（需确认） |
| `/结束` | 内建指令 | 保存状态并结束会话 |
| `/分析` | Skill | 需求分析和任务拆解 |
| `/设计` | Skill | 架构设计和技术选型 |
| `/视觉设计` | Skill | 海报、封面、PPT、UI 等视觉方案与评审 |
| `/测试` | Skill | 质量验证和排错 |
| `/部署` | Skill | 项目部署上线 |
| `/复盘` | Skill | 闭环总结优化 |
| `/项目归档` | Skill | 项目归档到 Builds/ |

内置 Skill 当前为：

- `分析`
- `设计`
- `视觉设计`
- `测试`
- `部署`
- `复盘`
- `项目归档`

#### 5. 记忆系统

框架会自动保存关键信息：

| 触发时机 | 保存内容 |
|----------|----------|
| 任务完成 | 更新 project_scope.md |
| 错误解决 | 记录到 knowledge/errors.md |
| 关键决策 | 记录到 knowledge/decisions.md |
| 发现偏好 | 更新 user_scope.md |
| 会话结束 | 生成 session-handoff.md + 长期记忆沉淀 |

#### 6. 闭环执行

当前版本不再把“能触发”当作“已完成”。核心流程统一要求：

- **预检**：先读取配置、记忆和相关上下文
- **执行**：必须真的改文件、跑诊断或执行命令
- **验证**：变更后进行 `GetDiagnostics`、命令验证或文件抽查
- **沉淀**：把关键结果写回 Memory / Evolve / 文档
- **回滚**：像 `/重置` 这类高风险流程必须有备份和失败恢复路径

## 规则体系

框架包含 7 个聚焦的规则文件：

| 规则文件 | 职责 |
|----------|------|
| `00-核心身份.md` | 身份定义、核心原则、禁止行为 |
| `01-记忆系统.md` | 记忆结构、加载/更新策略 |
| `02-技能调度.md` | 触发规则、编排模式 |
| `03-错误处理.md` | 4级错误分级、降级方案 |
| `04-会话管理.md` | 会话生命周期、恢复机制 |
| `05-自动保存.md` | 6个触发点、保存规范 |
| `06-输出规范.md` | 代码风格、对话风格 |

## 自定义

### 添加自定义 Skill

1. 在 `Skills/` 目录创建新文件夹（如 `Skills/我的技能/`）
2. 参考 [SKILL_TEMPLATE.md](.trae/documents/SKILL_TEMPLATE.md) 创建 `SKILL.md` 文件。
3. 也可以参考内置技能示例：[设计](.trae/skills/设计/SKILL.md)。

### 技能执行规范

- **触发即执行**：所有自定义技能必须包含 `## ⚠️ 执行声明` 章节，确保触发后立即进行实际工具操作。
- **元数据对齐**：YAML Frontmatter 中的 `name` 必须与子目录名一致，`trigger` 必须以 `/` 开头。
- **闭环对齐**：建议同步参考 `.trae/documents/闭环执行矩阵.md`，补齐预检、验收、失败处理和沉淀步骤。

### 修改规则

编辑 `.trae/rules/` 下的对应文件，保持 Markdown 表格格式。

## 注意事项

- 所有数据存储在本地，确保数据安全
- 遵循 `/指令` 机制，确保流程完整
- 生命周期指令优先由 Coordinator 执行，不强制建模为 Skill
- `Modules/` 不是当前阶段的默认目标，只有出现真实高复用资产时才建议沉淀
- 定期执行 `/复盘`，持续优化系统
- 框架是"活"的，会根据使用不断优化
