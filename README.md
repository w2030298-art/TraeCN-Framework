# Trae Native Agent Framework (TNAF)

> 🚀 一个专为 Trae 打造的“数字孪生”AI 编程助手框架。摒弃低效的纯文本记忆，全面拥抱 Trae 原生的 `manage_core_memory` 记忆库体系与 `Skill` 技能进程级调度机制。

## 为什么要使用 TNAF？

传统 AI 编程助手往往通过庞大的 Prompt 让 AI 扮演各种角色（如“你现在是前端架构师”），并用纯文本文件（如 `.md`）来维护长期的状态和记忆。
这种方式存在致命缺陷：
1. **记忆污染与衰退**：纯文本维护极易因为 AI 的上下文截断或脑补而产生内容丢失。
2. **多角色冲突**：在一个 Prompt 中赋予多个角色会让 AI 产生幻觉。
3. **效率低下**：大量的文字读写浪费 Token 且无法做到进程级任务隔离。

**本框架的优势：**
1. **原生记忆管理**：绝对禁止直接写入 `.md` 文件，所有偏好、踩坑记录强制使用 Trae 自带的 `manage_core_memory` 工具，实现结构化持久存储，彻底告别记忆丢失。
2. **多进程协作 (`Skill` 体系)**：把“角色扮演”下放到 Trae 的原生技能 (`Skills`) 系统中。主 Agent 负责任务调度（Coordinator），将需求分析、架构设计、测试部署委派给专门的子进程，避免上下文污染。
3. **原生工具集成**：深度集成 `TodoWrite` (任务规划追踪)、`SearchCodebase` (代码语义检索)、`RunCommand` (非阻塞命令执行) 等 Trae 强大工具，实现全闭环开发。

## 目录结构

```text
├── .trae/                    # 核心配置文件
│   ├── rules/                # 框架宪法与操作规范
│   │   ├── 00-Core-Architecture.md
│   │   └── agent.md          # 你的 Agent 核心人设设定
│   └── skills/               # 原生子进程技能
│       ├── Analysis.md       # 需求分析子进程
│       ├── Design.md         # 架构设计子进程
│       └── Session-Handoff.md# 会话交接与备份子进程
├── scripts/                  # 辅助自动化脚本 (备份/维护)
├── docs/                     # 架构文档与最佳实践
└── README.md                 # 本文件
```

## 快速开始

1. 将本项目的 `.trae` 文件夹复制到你的项目根目录。
2. 打开 `.trae/rules/agent.md`，将 `<your-agent-name>` 替换为你想要的助手名字。
3. 在 Trae 中开启新会话，你的专属数字孪生助手即可上线！

## 进化里程碑

本框架将帮助你的 Trae 助手实现以下进化：
- **Lv.1 规则驱动**：基于稳定的 rules 实现 95% 场景覆盖。
- **Lv.2 记忆驱动**：记忆库准确率 > 90%，不再需要重复提示你的技术栈和偏好。
- **Lv.3 预判驱动**：提前预判你的下一步需求，主动帮你创建组件或写好测试。
- **Lv.4 数字孪生**：你不用说，它就知道你需要什么。
