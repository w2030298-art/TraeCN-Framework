# TraeCN-Framework

> 专为 Trae IDE 打造的 AI 编程助手框架。基于 Read/Write 工具的结构化记忆 + Skill 技能调度。

[![Version](https://img.shields.io/badge/version-1.0.4-blue.svg)](https://github.com/qShan1/TraeCN-Framework)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

## 核心理念

传统方式用纯文本文件和庞大 Prompt 维护记忆和角色，存在记忆污染、多角色冲突、效率低下等问题。

本框架的方案：

- **结构化记忆**：Markdown 表格文件，易读、易更新、易搜索
- **Skill 技能调度**：通过 `/指令` 触发对应 Skill，任务隔离
- **Coordinator 模式**：主 Agent 负责任务拆解与调度，子任务委派给 Skill
- **开放衍生**：框架是"活"的，可以自我进化

## 目录结构

```
项目根目录/
├── .trae/              # 框架文件
│   ├── rules/          # 规则（7个聚焦文件）
│   ├── skills/         # 技能（10个内置技能）
│   └── documents/      # 设计文档
│
├── Core/记忆/          # 记忆文件
│   ├── user_scope.md   # 用户画像
│   ├── project_scope.md # 项目状态
│   ├── session-handoff.md # 会话交接
│   └── knowledge/      # 知识库
│       ├── errors.md   # 错误记录
│       ├── patterns.md # 设计模式
│       └── decisions.md # 关键决策
│
├── Builds/             # 已完成项目
├── Skills/             # 自定义技能
├── Evolve/             # 进化日志、错题本
└── Backup/             # 备份
```

## 快速开始

### 1. 导入框架

将 `.trae/` 文件夹复制到你的项目根目录。

### 2. 初始化

在 Trae IDE 中打开工作区，在聊天框输入：

```
/初始化
```

AI 助手将引导你完成：

- 创建目录结构
- 生成空白记忆文件
- 配置用户偏好

### 3. 使用

通过 `/指令` 触发技能：

| 指令 | 功能 |
|------|------|
| `/初始化` | 新用户首次使用 |
| `/重置` | 清除用户数据（需确认） |
| `/分析` | 需求分析和任务拆解 |
| `/设计` | 架构设计和技术选型 |
| `/测试` | 质量验证和排错 |
| `/部署` | 项目部署上线 |
| `/复盘` | 闭环总结优化 |
| `/项目归档` | 项目归档到 Builds/ |
| `/结束` | 保存状态并结束会话 |

### 4. 记忆系统

框架会自动保存关键信息：

| 触发时机 | 保存内容 |
|----------|----------|
| 任务完成 | 更新 project_scope.md |
| 错误解决 | 记录到 knowledge/errors.md |
| 关键决策 | 记录到 knowledge/decisions.md |
| 发现偏好 | 更新 user_scope.md |
| 会话结束 | 生成 session-handoff.md |

## 规则体系

框架包含 7 个聚焦的规则文件：

| 规则文件 | 职责 |
|----------|------|
| `00-核心身份.md` | 身份定义、核心原则、禁止行为 |
| `01-记忆系统.md` | 记忆结构、加载/更新策略 |
| `02-技能调度.md` | 触发规则、编排模式 |
| `03-错误处理.md` | 4级错误分级、降级方案 |
| `04-会话管理.md` | 会话生命周期、恢复机制 |
| `05-自动保存.md` | 6个触发点、保存规范 |
| `06-输出规范.md` | 代码风格、对话风格 |

## 自定义

### 添加自定义 Skill

1. 在 `Skills/` 目录创建新文件夹
2. 创建 `SKILL.md` 文件，格式：

```markdown
---
name: 技能名
description: 一句话描述
trigger: /触发词
---

# 技能名

## 触发条件
什么时候触发

## 执行步骤
1. 第一步
2. 第二步

## 输出
产生什么结果
```

### 修改规则

编辑 `.trae/rules/` 下的对应文件，保持 Markdown 表格格式。

## 注意事项

- 所有数据存储在本地，确保数据安全
- 遵循 `/指令` 机制，确保流程完整
- 定期执行 `/复盘`，持续优化系统
- 框架是"活"的，会根据使用不断优化
