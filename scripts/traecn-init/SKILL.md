# TraeCN-Framework 初始化引导技能

## ⚠️ 执行声明
本技能被触发时，**必须实际执行**以下流程（使用 Read/Write/RunCommand/Glob/LS 等工具），**不得仅输出文档内容而不执行实际操作**。

## 闭环要求
默认遵循 `.trae/documents/闭环执行矩阵.md` 的五段式闭环：预检 -> 执行 -> 诊断 -> 验证 -> 沉淀。若 `.trae/` 尚未存在，则沉淀步骤在部署完成后执行。

## 触发条件
- 用户输入 `/初始化`（无参数）时。
- 用户输入 `/初始化 <http/https-url>` 时。
- 用户输入 `/初始化 @<索引标识>` 时。

## 参数解析规则
1. **无参数** (`/初始化`)：检测工作区 `.trae/` 是否存在，存在则标准初始化，不存在则自动部署后初始化。
2. **URL** (`/初始化 https://...`)：从指定 URL 下载配置包，校验后部署。
3. **本地索引** (`/初始化 @default`)：从本地配置索引读取并部署。

## 预检阶段

### 步骤 1：解析输入
- 提取用户输入中 `/初始化` 后的参数。
- 判断模式：无参数 / URL / @索引。

### 步骤 2：检测当前环境
- 使用 `Glob` 或 `LS` 检查工作区根目录是否存在 `.trae/` 目录。
- 使用 `RunCommand` 检查网络连通性（如需下载）：`Test-Connection -ComputerName github.com -Count 1 -Quiet`

### 步骤 3：权限预检
- 尝试在工作区根目录创建临时文件 `.trae-init-test.tmp`，确认写入权限。
- 创建成功后立即删除该临时文件。

## 执行阶段

### 分支判断

```
/初始化 [参数]
  ├─ .trae/ 已存在 → 跳到【标准初始化】
  └─ .trae/ 不存在
       ├─ 有 URL 参数 → 【方式一：链接导入】
       ├─ 有 @索引参数 → 【方式二：本地索引】
       └─ 无参数 → 【方式三：快捷自动】
```

---

### 【方式一：链接导入初始化】

**触发**：`/初始化 <url>`

**执行步骤**：
1. **验证 URL**：确认参数以 `http://` 或 `https://` 开头，否则报错终止。
2. **下载**：使用 `Invoke-WebRequest` 下载到 `$env:TEMP\traecn-init-下载\`。
   - 命令示例：`Invoke-WebRequest -Uri "<url>" -OutFile "$env:TEMP\traecn-init-download.zip"`
3. **解压**：若文件为 `.zip`，使用 `Expand-Archive` 解压。
   - 若 URL 指向 GitHub 仓库 zip，解压后进入 `<repo>-main/` 子目录。
4. **定位 .trae/**：在解压后的目录中查找 `.trae/` 文件夹。
5. **完整性校验**：执行【校验清单】。
6. **部署**：将 `.trae/` 复制到工作区根目录。
7. **清理**：删除临时下载文件。
8. **标准初始化**：执行【标准初始化】。

**错误处理**：
- URL 格式无效：报错并提示正确格式。
- 下载失败：提示检查网络/URL，建议手动下载。
- 解压失败：提示格式不支持，建议手动解压后复制。
- 找不到 `.trae/`：提示链接内容可能不是 TraeCN 配置。
- 校验失败：列出缺失文件，终止并建议检查来源。

---

### 【方式二：本地索引初始化】

**触发**：`/初始化 @<索引标识>`

**执行步骤**：
1. **读取索引**：读取 `$env:USERPROFILE\.trae-configs\index.md`。
2. **匹配索引**：在表格中查找用户指定的索引标识。
3. **定位配置**：根据索引中的路径定位 `.trae/` 目录。
   - 路径为相对路径时，基于 `$env:USERPROFILE\.trae-configs\` 解析。
4. **完整性校验**：执行【校验清单】。
5. **部署**：将配置复制到工作区根目录。
6. **标准初始化**：执行【标准初始化】。

**首次使用引导**：
- 若索引目录不存在，自动创建并提示用户如何添加配置。
- 若索引文件不存在，生成默认模板并提示。

**错误处理**：
- 索引标识未找到：列出可用索引，建议检查拼写。
- 配置路径不存在：提示配置可能已被移动，建议更新索引。
- 校验失败：列出缺失文件。

---

### 【方式三：快捷自动初始化】

**触发**：`/初始化`（无参数，且工作区无 `.trae/`）

**执行步骤**：
1. **查找本地默认配置**：检查 `$env:USERPROFILE\.trae-configs\default\.trae\` 是否存在。
2. **本地有默认配置** → 复制部署 → 执行【标准初始化】。
3. **本地无默认配置** → 从 GitHub 官方仓库下载：
   - 默认源：`https://github.com/w2030298-art/TraeCN-Framework/archive/refs/heads/main.zip`
   - 下载到 `$env:TEMP\traecn-init-download.zip`
   - 解压后进入 `TraeCN-Framework-main\.trae\`
   - 复制到工作区根目录
   - 清理临时文件
   - 执行【标准初始化】

**错误处理**：
- GitHub 不可达：提示网络问题，建议手动下载或配置本地索引。
- 下载/解压失败：降级为提示手动操作步骤。

## 校验清单

部署 `.trae/` 前必须执行以下检查：

| 检查项 | 必须 | 检查方式 |
|--------|------|----------|
| `.trae/config.md` 存在 | ✅ 必须 | `Glob .trae/config.md` |
| `.trae/rules/` 目录存在 | ✅ 必须 | `LS .trae/rules/` |
| `.trae/rules/00-核心身份.md` 存在 | ✅ 必须 | `Read .trae/rules/00-核心身份.md` |
| `.trae/skills/` 目录存在 | ⚠️ 建议 | `LS .trae/skills/` |
| `.trae/documents/` 目录存在 | ⚠️ 建议 | `LS .trae/documents/` |

- 缺少必须项 → **报错终止**，不继续初始化。
- 缺少建议项 → **警告但继续**，初始化后提示补充。

---

## 标准初始化

在 `.trae/` 部署完成（或已存在）后执行：

1. **创建目录结构**：
   - `Core/记忆/`
   - `Core/记忆/knowledge/`
   - `Evolve/`
   - `Backup/`
   - `Skills/`

2. **创建/检查记忆文件**（缺失则写入最小模板，不覆盖已有内容）：
   - `Core/记忆/user_scope.md`
   - `Core/记忆/project_scope.md`
   - `Core/记忆/session-handoff.md`
   - `Core/记忆/knowledge/errors.md`
   - `Core/记忆/knowledge/decisions.md`
   - `Core/记忆/knowledge/patterns.md`

3. **检测旧会话**：
   - 若 `session-handoff.md` 存在未完成事项，提示用户是否继续。

4. **读取配置**：
   - 读取 `.trae/config.md`，确认开关状态。

---

## 诊断阶段

1. **文件写入诊断**：使用 `Glob` 确认 `.trae/` 已完整复制到工作区。
2. **记忆文件诊断**：`Read` 检查 `Core/记忆/user_scope.md` 和 `project_scope.md` 是否可读。
3. **命令诊断**：若使用了下载/解压命令，检查命令退出码和输出。

## 验证阶段

1. **结构验证**：确认 `.trae/` 包含 `config.md`、`rules/`、`skills/`、`documents/`。
2. **文件验证**：随机读取 1-2 个规则文件确认内容正确。
3. **记忆验证**：确认 `Core/记忆/` 下关键文件存在且可读。
4. **开关验证**：从 `config.md` 确认 Global Hooks 等开关状态。

## 沉淀阶段

1. **更新 project_scope.md**：记录初始化方式、时间和当前状态。
2. **输出初始化摘要**：
   - 初始化方式（URL/本地/自动下载）
   - 已创建/已验证的目录和文件
   - 当前 `.trae/config.md` 开关状态
   - 下一步建议

---

## 输出要求

- **已完成动作**：实际做了哪些下载、复制、创建、验证操作。
- **验证结果**：`.trae/` 结构是否完整，记忆文件是否可读。
- **当前状态**：`.trae/` 是否存在，`Core/记忆/` 是否就绪，有无旧会话。
- **下一步建议**：用户此刻最值得做的 1-3 步。

## 验收标准

- `.trae/config.md` 存在且可读。
- `.trae/rules/` 至少包含 `00-核心身份.md`。
- `Core/记忆/` 下关键记忆文件存在且可读。
- 回复中包含明确的初始化摘要和下一步建议。

## 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 工作区写入权限不足 | 提示以管理员身份运行 Trae，或检查目录权限 |
| 下载失败（网络/URL） | 提示手动下载并复制 `.trae/`，或配置本地索引 |
| `.trae/` 已存在 | 提示已存在，提供覆盖/合并/取消选项，默认取消 |
| 校验失败（缺少必须文件） | 列出缺失文件，终止并建议检查来源 |
| 解压失败 | 提示格式不支持，建议手动解压 |
| GitHub 不可达 | 降级为提示配置本地 `@default` 索引 |
| 旧会话未完成任务 | 提示恢复入口，由用户选择继续或新任务 |

## 禁止行为

- 未经确认覆盖已有的 `.trae/` 目录。
- 在校验失败时继续执行标准初始化。
- 下载失败后不提示替代方案。
- 仅输出说明而不执行实际文件操作。

## 技术备注

- Windows 环境下使用 PowerShell 命令进行下载和解压。
- 临时文件统一放在 `$env:TEMP\traecn-init-*` 目录下，完成后清理。
- 本地索引根目录：`$env:USERPROFILE\.trae-configs\`
- 默认远程源：`https://github.com/w2030298-art/TraeCN-Framework/archive/refs/heads/main.zip`
- 所有路径使用绝对路径或 `$env:` 环境变量，避免相对路径歧义。
