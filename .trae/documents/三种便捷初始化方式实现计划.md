# 三种便捷初始化方式实现计划

## 背景分析

当前 TraeCN-Framework 的初始化流程存在一个关键缺口：**用户必须手动复制 `.trae/` 文件夹到项目根目录**，然后才能执行 `/初始化`。这意味着：

1. `/初始化` 仅负责创建 `Core/记忆/` 等运行态目录，不负责部署 `.trae/` 本身
2. 没有自动化获取 `.trae/` 的手段
3. 新用户上手门槛高（需手动下载/复制）

本次需求将补齐 `.trae/` 部署环节，使 `/初始化` 成为真正的"一键启动"入口。

## 设计原则

- **声明式实现**：遵循框架现有模式，通过规则文件和指令文档定义执行逻辑，由 Coordinator 在运行时解释执行
- **渐进增强**：三种方式分层递进——链接导入（最灵活）→ 本地索引（最快速）→ 快捷指令（最简单）
- **统一入口**：所有方式都通过 `/初始化` 指令触发，通过参数区分模式
- **闭环保障**：每种方式都遵循"预检→执行→诊断→验证→沉淀"五段式闭环

## 三种初始化方式设计

### 方式一：链接导入初始化

**触发格式**：`/初始化 <http/https-url>`

**示例**：
- `/初始化 https://github.com/qShan1/TraeCN-Framework/archive/refs/heads/main.zip`
- `/初始化 https://example.com/my-trae-config.zip`

**执行流程**：
1. **预检**：验证 URL 格式（http/https），检测工作区是否已存在 `.trae/`（存在则提示覆盖/合并/取消）
2. **下载**：使用 `Invoke-WebRequest`（PowerShell）下载远程资源到临时目录
3. **解压**：若为 zip/tar.gz 等压缩包则自动解压；若为 Git 仓库则提取其中的 `.trae/` 目录
4. **完整性校验**：检查下载内容是否包含 `.trae/` 必要结构（`config.md`、`rules/`、`skills/`、`documents/`）
5. **部署**：将 `.trae/` 复制到工作区根目录
6. **清理**：删除临时下载文件
7. **后续初始化**：执行标准 `/初始化` 流程（创建 `Core/记忆/` 等）
8. **验证**：读取关键文件确认可读，输出初始化摘要

**完整性校验标准**：
- 必须存在：`.trae/config.md`、`.trae/rules/`（至少含 `00-核心身份.md`）
- 建议存在：`.trae/skills/`、`.trae/documents/`
- 缺少必须文件则报错并终止，缺少建议文件则警告但继续

**错误处理**：
- 下载失败：检查网络/URL 有效性，建议检查链接或重试
- 解压失败：提示格式不支持，建议手动下载
- 校验失败：列出缺失文件，建议检查链接内容是否为有效 TraeCN 配置
- 部署失败（权限等）：建议检查工作区写入权限

### 方式二：本地配置索引初始化

**触发格式**：`/初始化 @<索引标识>`

**示例**：
- `/初始化 @default` — 使用默认配置
- `/初始化 @web-project` — 使用名为 web-project 的本地配置
- `/初始化 @minimal` — 使用最小化配置

**索引存储机制**：
- 索引根目录：`%USERPROFILE%\.trae-configs\`（Windows）或 `$HOME/.trae-configs/`（跨平台）
- 索引文件：`%USERPROFILE%\.trae-configs\index.md`
- 配置存储：每个索引对应一个子目录，如 `%USERPROFILE%\.trae-configs\web-project\.trae\`

**索引文件格式**（`index.md`）：
```markdown
# TraeCN 本地配置索引

| 索引标识 | 配置路径 | 说明 | 更新时间 |
| :--- | :--- | :--- | :--- |
| default | ~/.trae-configs/default/ | 官方默认完整配置 | YYYY-MM-DD |
| minimal | ~/.trae-configs/minimal/ | 最小化配置（仅核心规则） | YYYY-MM-DD |
| web-project | ~/.trae-configs/web-project/ | Web 项目专用配置 | YYYY-MM-DD |
```

**索引命名规范**：
- 仅允许小写字母、数字、连字符（`-`）
- 长度 2-30 个字符
- 保留标识：`default`（官方默认）、`minimal`（最小化）
- 禁止使用：`..`、`/`、`\`、空格

**执行流程**：
1. **预检**：检测索引根目录是否存在，不存在则创建并写入默认索引
2. **查找索引**：读取 `index.md`，匹配用户指定的索引标识
3. **定位配置**：根据索引中的路径定位 `.trae/` 目录
4. **完整性校验**：同方式一的校验标准
5. **部署**：将配置复制到工作区根目录
6. **后续初始化**：执行标准 `/初始化` 流程
7. **验证**：读取关键文件确认可读，输出初始化摘要

**首次使用自动引导**：
- 索引目录不存在时，自动创建并提示用户如何添加配置
- 提供 `@default` 的自动生成选项（从 GitHub 下载官方配置到本地索引）

**错误处理**：
- 索引标识未找到：列出可用索引，建议检查拼写
- 配置路径不存在：提示配置可能已被移动，建议更新索引
- 校验失败：同方式一

### 方式三：快捷指令自动初始化

**触发格式**：`/初始化`（无参数）

**行为逻辑**：
```
/初始化
  ├─ .trae/ 已存在 → 执行标准初始化（创建 Core/记忆/ 等）
  └─ .trae/ 不存在 → 自动部署 .trae/ 后执行标准初始化
       ├─ 本地索引有 @default → 从本地索引部署
       └─ 本地索引无 @default → 从 GitHub 官方仓库下载部署
```

**自动部署源优先级**：
1. 本地索引 `@default`（最快，离线可用）
2. GitHub 官方仓库最新 Release（需网络）
3. GitHub main 分支（降级方案）

**执行流程**：
1. **预检**：检测 `.trae/` 是否已存在
2. **分支判断**：
   - 已存在 → 直接跳到步骤 6
   - 不存在 → 继续步骤 3
3. **查找本地默认配置**：检查 `%USERPROFILE%\.trae-configs\default\`
4. **本地有默认配置** → 复制部署；**本地无** → 从 GitHub 下载
5. **完整性校验**：同方式一
6. **标准初始化**：创建 `Core/记忆/`、`knowledge/`、`Evolve/`、`Backup/`、`Skills/` 等目录和模板文件
7. **验证**：读取关键文件确认可读
8. **输出摘要**：初始化方式、已创建文件、当前状态、下一步建议

**权限与依赖检查**：
- 检测工作区写入权限（尝试创建临时文件验证）
- 网络下载时检测网络连通性
- PowerShell 执行策略检查（如需执行脚本）

**错误处理**：
- 本地默认配置损坏：降级为 GitHub 下载
- GitHub 不可达：提示网络问题，建议手动下载或配置本地索引
- 权限不足：建议以管理员身份运行或检查目录权限

## 风险与降级

| 风险 | 影响 | 降级方案 |
| --- | --- | --- |
| GitHub 不可达 | 快捷指令无法自动下载 | 降级为提示手动下载，或使用本地索引 |
| 压缩包格式多样 | 链接导入解压失败 | 支持 zip/tar.gz，其他格式提示手动解压 |
| 本地索引损坏 | 索引初始化失败 | 重建索引模板，提示用户重新配置 |
| .trae/ 已存在冲突 | 覆盖风险 | 默认不覆盖，提供覆盖/合并/取消选项 |
