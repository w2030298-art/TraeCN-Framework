# 视觉设计调用示例

这份文档用于告诉使用者：什么样的输入适合调用 `视觉设计`，以及期望得到什么样的输出结构。

## 示例 1：文章封面

### 用户输入
```md
/视觉设计
帮我做一个 AI 编程助手相关文章封面的视觉方案，发在公众号，想要专业但别太冷。
```

### 期望输出
- 需求重述
- 场景与约束
- 3 个差异化方向
- 推荐主方向
- 配色建议
- 构图和文字层级建议
- AI 出图提示词
- 设计评审
- 验证范围与风险

## 示例 2：PPT 首页与目录页

### 用户输入
```md
/视觉设计
我要做一份给投资人看的 AI 产品介绍 PPT，先给我首页和目录页的视觉方向，偏高级、可信、克制。
```

### 期望输出
- 目标受众与场景判断
- 场景与约束
- 3 个适合 PPT 的方向
- 首页排版建议
- 目录页层级建议
- 配色、字体、留白规则
- 投影场景下的风险提示

## 示例 3：App UI 原型

### 用户输入
```md
/视觉设计
帮我设计一个任务管理 App 的首页 UI 风格，用户是年轻职场人，我想要清爽但不要幼稚。
```

### 期望输出
- 用户与使用场景重述
- 场景与约束
- 3 个 UI 方向
- 主界面布局说明
- 颜色、字号、间距和组件气质建议
- 交互注意点
- 可继续交给设计工具实现的结构描述
- 验证范围与风险

## 示例 4：现有视觉稿评审

### 用户输入
```md
/视觉设计
我已经有一版海报了，你帮我做设计评审，重点看层级、配色和传播力。
```

### 期望输出
- 评审前提说明
- 场景与约束
- 五维评分
- Keep
- Fix
- Quick Wins
- 是否建议重做方向的判断
- 验证范围与风险

## 示例 5：不显式写 `/视觉设计`

### 用户输入
```md
帮我做一个发布会海报，黑金、高级、压迫感一点。
```

### 系统应如何处理
- 因命中“海报”与视觉风格诉求，直接按 `视觉设计` 流程处理。
- 若信息不足，只补问目标受众、发布场景、输出规格这 1-3 个关键问题。
