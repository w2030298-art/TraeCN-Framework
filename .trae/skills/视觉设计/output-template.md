# 视觉设计输出模板

用于在 `视觉设计` 执行时快速产出稳定、完整、可比较的回答结构。

## 标准模板

```md
## 需求重述
- 这次设计要解决什么问题
- 服务谁
- 用在什么场景
- 本次重点解决什么，不解决什么

## 场景与约束
- 载体：
- 输出规格：
- 核心信息：
- 情感基调：
- 关键限制：

## 方向一：方向名称
- 为什么适合
- 核心特征
- 气质关键词
- 适用场景
- 风险点

## 方向二：方向名称
- 为什么适合
- 核心特征
- 气质关键词
- 适用场景
- 风险点

## 方向三：方向名称
- 为什么适合
- 核心特征
- 气质关键词
- 适用场景
- 风险点

## 推荐方向
- 推荐哪一个
- 为什么
- 如需混合，混合哪些元素

## 执行产物

### 配色方案
- 主色：
- 辅色：
- 强调色：
- 中性色：

### 版式与层级
- 标题层级
- 正文层级
- 留白策略
- 构图建议

### 提示词 / 草图说明 / 页面结构
- 如果走 AI 出图，给提示词
- 如果走界面设计，给页面结构
- 如果走海报封面，给构图草图说明

## 设计评审
- 哲学一致性：X/10
- 视觉层级：X/10
- 细节执行：X/10
- 功能性：X/10
- 创新性：X/10

### Keep
- 保留项

### Fix
- 问题 -> 原因 -> 修复建议

### Quick Wins
- 5 分钟优先修的 3 件事

## 验证范围与风险

- 残留风险：

## 下一步建议
- 继续深化方向 / 进入出图 / 进入实现 / 继续评审
```

## 使用原则

- 需要完整方案时，优先使用标准模板。
- 需求简单或用户只想先看方向时，可降级为简化模板。
- 无论使用哪个模板，都不能跳过“可比较方向”“设计评审”和“验证范围与风险”。
