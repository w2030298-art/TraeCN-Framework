## 场景模板

用于把视觉需求快速映射到规格、信息密度和推荐风格，避免只谈审美不谈落地。

### 1. 封面 / 海报
- 常用规格：
  - 横版封面：`1200x675`
  - 竖版海报：`1080x1920`
  - 方形社媒：`1080x1080`
- 重点：
  - 1 个核心视觉焦点
  - 标题强层级，避免堆字
  - 色彩和构图要先抢到注意力
- 推荐风格：`Sagmeister`、`Kenya Hara`、`Takram`

### 2. 正文配图 / 概念插画
- 常用规格：
  - `16:9`
  - `4:3`
  - `1:1`
- 重点：
  - 服务观点，而不是装饰
  - 只表达 1 个核心概念
  - 与文章语气一致
- 推荐风格：`Takram`、`Fathom`、`Kenya Hara`

### 3. 信息图 / 数据可视化
- 常用规格：
  - 竖版长图：`1080x1920`
  - 横版演示：`1920x1080`
  - 方形卡片：`1080x1080`
- 重点：
  - 标题 -> 核心数据 -> 细节说明 三层结构
  - 图表准确，不拿装饰替代信息
  - 视觉引导线清楚
- 推荐风格：`Fathom`、`Pentagram`

### 4. PPT / Keynote
- 常用规格：
  - `16:9`
  - `16:10`
- 重点：
  - 每页只讲 1 个核心点
  - 标题、正文、注释层级明显
  - 投影场景下要留白更多
- 推荐风格：`Pentagram`、`Kenya Hara`、`Fathom`

### 5. PDF / 白皮书 / 技术报告
- 常用规格：
  - `A4`
  - `Letter`
- 重点：
  - 长文阅读节奏
  - 清晰导航、章节、页码系统
  - 图表与正文共存，不互相抢夺
- 推荐风格：`Fathom`、`Pentagram`

### 6. 官网 / 落地页
- 常用规格：
  - Desktop 设计宽度：`1440`
  - Mobile 最小关注：`375`
- 重点：
  - 首屏 5 秒讲清价值
  - CTA 清楚
  - 支持滚动叙事和移动端适配
- 推荐风格：`Locomotive`、`Pentagram`、`Takram`

### 7. App UI / 原型界面
- 常用规格：
  - iPhone：`390x844`
  - Android：`360x800`
  - iPad：`1024x1366`
- 重点：
  - 触控区域最少 `44x44`
  - 信息密度适中
  - 优先保证可用性，而不是装饰感
- 推荐风格：`Takram`、`Pentagram`

## 使用方法

- 先判断用户要的是什么载体，再从本文件取规格和重点。
- 再到 `design-styles.md` 里选 3 个方向做差异化推荐。
- 最后用 `critique-guide.md` 做评审，不跳过验收。
