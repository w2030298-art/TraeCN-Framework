# 复盘技能

## ⚠️ 执行声明
本技能被触发时，**必须实际执行**以下所有步骤（使用 Read/Write 工具操作文件），
**不得仅输出本文档内容而不执行实际操作**。

## 功能说明
当用户输入 `/复盘` 或 `总结` 触发词时，执行完整的复盘流程。

## 执行步骤

### 第一步：复盘
1. 回顾本次任务的目标、流程、关键决策
2. 识别踩坑点、错误、卡点
3. 提炼成功的关键因素
4. 记录本次使用的技能类型

### 第二步：沉淀
更新以下文件：
- `Core/记忆/user_scope.md` — 新增本次偏好、技术栈变化、编码习惯更新
- `Core/记忆/project_scope.md` — 标记本次关键上下文
- `Core/记忆/knowledge/errors.md` — 新增踩坑记录+解决方案（如有）
- `Core/记忆/knowledge/decisions.md` — 新增关键决策记录（如有）
- `Core/记忆/knowledge/patterns.md` — 新增设计模式积累（如有）
- `Evolve/错题本.md` — 新增踩坑记录+解决方案（如有）
- `Evolve/进化日志.md` — 新增本次任务复盘摘要

### 第三步：复用与资产化
1. **识别可复用组件**：识别本次产生的原子化逻辑，抽离至 `Modules/` 对应子目录。
2. **更新组件索引**：在 `Modules/README.md` 中登记新增资产。
3. **沉淀偏好**：更新记忆库中的技术栈偏好。
