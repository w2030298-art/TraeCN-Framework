# 自动保存

## 核心理念
用户不需要记住"保存"，所有关键信息自动沉淀到记忆文件。

## 触发点
| 触发条件 | 执行动作 |
|----------|----------|
| 任务完成 | 更新 project_scope.md（完成功能+下一步） |
| 错误解决 | 写入 knowledge/errors.md |
| 做出决策 | 写入 knowledge/decisions.md |
| 发现偏好 | 更新 user_scope.md |
| 执行 `/复盘` | 更新 Evolve/进化日志.md 和 Evolve/错题本.md |
| 轻量结束会话（如“先暂停”“下次继续”） | 优先更新 session-handoff.md，写入待继续状态与恢复入口 |
| 标准结束会话（如“结束会话”或 `/结束`） | 由 Coordinator 生成 session-handoff.md + 长期记忆沉淀 |
| 执行 `/重置` 前 | 先备份 Core/记忆 到 Backup/reset-时间戳/ |
| "记住这个" | 立即保存到 user_scope.md |
| **自愈成功** | 将修复方案记录到 `Evolve/错题本.md` 以防复发 |
| **代码模式发现** | 将高质量实现模式提取至 `knowledge/patterns.md` |

## 保存规范
- 单次写入不超过 50 行
- 使用 Markdown 表格
- 添加时间戳（updated: YYYY-MM-DD）
- 保持格式一致

## 流程沉淀矩阵

| 流程 | 必写文件 | 可选补充 |
| --- | --- | --- |
| `/初始化` | `project_scope.md` | `user_scope.md`（若确认了偏好） |
| `/重置` | `session-handoff.md`（如需记录失败/回滚） | `knowledge/errors.md` |
| 轻量结束会话 | `session-handoff.md` | `project_scope.md`、`knowledge/decisions.md` |
| 标准结束会话（系统命令：`/结束`） | `session-handoff.md`、`project_scope.md` | `user_scope.md`、`knowledge/decisions.md` |
| `/复盘` | `Evolve/进化日志.md`、`project_scope.md` | `knowledge/errors.md`、`patterns.md`、`decisions.md` |
| 自愈修复 | `knowledge/errors.md` | `Evolve/错题本.md`、`patterns.md` |

要求：
- 写入前先判断是否真的产生了新信息，避免空洞重复记录
- 写入后应抽查读取结果，确认表格结构未损坏

## 自动清理
- 同类错误保留最新 5 条
- 超过 30 天未更新的项目提示归档
- 会话交接继续后清空

## 主动性 (Proactivity & Cascade Mode)
Coordinator 必须根据当前状态主动提供建议，避免用户盲目输入：
- **感知修复**：当检测到 Linter 错误或终端报错时，**立即主动调用 `GetDiagnostics` 或分析报错**，并给出修复方案，严禁等待用户告知报错。
- **复杂错误解决后**：主动建议执行 `/复盘` 并记录到 `Evolve/错题本.md`。
- **检测到新模块/大变更**：主动建议执行 `/分析` 或 `/设计` 以同步 `project_scope.md`。
- **任务队列为空时**：主动根据 `project_scope.md` 的“下一步”引导用户。
- **发现规则冲突时**：主动建议更新 `.trae/rules/`。
- **流程只触发未闭环时**：主动补齐验收、验证、沉淀或回滚步骤，不能把“已触发”视为“已完成”。
