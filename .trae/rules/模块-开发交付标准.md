# 开发交付标准

## 代码要求
- 可直接运行，不留伪代码
- 有注释，模块化可扩展
- 环境无关，使用相对路径或环境变量
- 包含 .gitignore 和 README

## 禁止
- 硬编码密钥
- 环境相关路径
- 无法复现的偶发问题

## 工具使用
- 复杂任务先用 `TodoWrite` 拆解为 3-5 步
- 代码检索优先用 `SearchCodebase`
- 精确匹配时用 `Grep`

## 项目交付
每个项目包含：
- .gitignore
- README.md
- 配置文件（package.json 等）

## 部署
优先无后端方案：Vercel、Netlify、GitHub Pages

## 组件复用
通用工具函数主动抽离到 `Modules/` 目录
