# 多 Agent 原生协作规则

## 核心理念
不再依赖纸面设定的"伪多Agent"系统，而是基于 Trae 的原生 `Skill` 工具实现**真正的进程级调度**。

## 协作流程（真实 Tool 驱动）

### 1. 需求分析阶段
- **触发条件**：用户提出模糊的新需求或宏大目标。
- **执行动作**：主Agent调用 `call:default_api:Skill{name="分析"}`。
- **结果**：获取需求拆解和复杂度评估。

### 2. 架构设计阶段
- **触发条件**：需要进行技术选型、目录设计、数据库设计。
- **执行动作**：主Agent调用 `call:default_api:Skill{name="设计"}`。
- **结果**：获取系统设计方案。

### 3. 开发阶段
- **触发条件**：主Agent作为Coordinator，根据任务性质调用相关开发技能，或者直接执行明确的编码指令。

### 4. 测试与排错
- **触发条件**：代码报错、编译失败、需要补充测试用例。
- **执行动作**：主Agent调用 `call:default_api:Skill{name="测试"}`。

### 5. 部署
- **触发条件**：项目完成，需要发布上线。
- **执行动作**：调用 `call:default_api:Skill{name="部署"}`。

## 冲突与容错
- 任何子 Agent（Skill）返回失败或无意义结果时，主 Agent 需要接管上下文，调整参数重新调用或切换方案，严禁直接把报错甩给用户。
