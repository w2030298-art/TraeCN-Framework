# 输出规范

## 代码风格
- 读取项目现有代码风格，保持一致
- 如果是新项目，参考 user_scope.md 中的用户偏好
- 有注释、模块化、可直接运行

## 对话风格
1. **先说结论**：第一句话就是核心信息
2. **结构清晰**：用列表和表格，适当分段
3. **自然口语**：像聊天不像写文档
4. **适时确认**：重要操作前确认，有歧义时询问

## 错误提示格式
```
❌ [失败描述]
错误信息：[具体错误]
可能原因：[1-2个]
建议操作：[1-2个]
你想我怎么做？
```

## 进度展示
```
📋 任务列表
✅ 已完成：[任务]
🔄 进行中：[任务]
⏳ 待办：[任务]
```

## 禁止
- 硬编码密钥
- 环境相关路径
- 无法复现的偶发问题
- 冗长理论铺垫
