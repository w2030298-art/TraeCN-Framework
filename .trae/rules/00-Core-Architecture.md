# 原生核心架构铁则（最高宪法）

## 1. 记忆归一（绝对禁止写.md记忆文件）
- **严禁**：手动读取或写入任何以 `.md` 结尾的记忆库文件（如 `/docs/记忆.md` 等）来保存长期状态。
- **必须**：发现用户的新偏好、新习惯、新踩坑记录时，**唯一合法操作**是调用原生 `manage_core_memory` 工具，执行 `ADD` 或 `UPDATE`。
- **作用域**：
  - `<user_scope>`：记录跨项目通用的习惯、踩坑、技术栈。
  - `<project_scope>`：记录当前项目的具体状态、架构设计。
- **防爆满机制**：当条目接近 20 条时，**必须优先使用 `UPDATE`** 将同类信息整合到已有记忆中，避免冗余。

## 2. 实体化 Agent（严禁脑补角色）
- **严禁**：在主会话中假装自己是“前端工程师”或“测试人员”并直接执行大段代码。
- **必须**：通过 `call:default_api:Skill` 调用对应的原生技能（如 `Analysis`, `Design`, `Test`）。
- **执行逻辑**：主Agent负责任务拆解与调度（Coordinator），将子任务委派给相应的 Skill 执行，等待 Skill 返回结果后再整合输出。

## 3. 工具至上（强制使用原生 Trae 能力）
- **任务规划**：接到复杂任务时，**第一步必须调用 `TodoWrite` 工具** 拆解任务步骤，并在执行中持续更新状态，让用户清晰看到进度。
- **代码检索**：禁止盲目猜测，必须使用 `SearchCodebase` 进行全局语义检索；使用 `Grep` 进行精确文本匹配。
- **状态验证**：Web项目开发中，启动服务（`RunCommand`，非阻塞）后，必须使用 `OpenPreview` 为用户打开预览面板。
- **交互确认**：遇到多选或不确定方案时，必须调用 `AskUserQuestion` 弹出选项卡让用户选择，而不是在聊天框里让用户打字回复。

## 4. 自动兜底与会话交接
- **触发交接**：当用户准备结束工作或挂起当前任务时，强制调用 `Session-Handoff` 技能。
- **执行逻辑**：将会话摘要写入本地 `交接文档.md`，并在下次启动时主动读取，实现跨会话无缝衔接。
