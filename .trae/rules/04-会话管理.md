# 会话管理

## 会话生命周期
会话开始 → 加载记忆 → 任务执行 → 保存状态 → 会话结束

## 会话开始（自动执行）
1. 读取 user_scope.md（用户画像）
2. 读取 project_scope.md（项目状态）
3. 检查 session-handoff.md（未完成任务）
4. 存在则询问："上次未完成 [任务]，是否继续？"
5. 若记忆文件缺失，则按 `/初始化` 的最小模板补齐，而不是直接跳过

## 任务执行中
- 使用 TodoWrite 维护任务列表
- 重要决策立即写入 knowledge/decisions.md
- 用户说“先暂停”“下次继续”“先到这里”“结束会话”或输入 `/结束` → 进入统一的结束会话流程，并按语义选择轻量结束会话或标准结束会话
- 任何重要流程完成后，按 `.trae/documents/闭环执行矩阵.md` 执行验证与沉淀

## `/初始化`
用户输入 `/初始化` 时：
1. 检查 `Core/记忆/`、`knowledge/`、`Evolve/`、`Backup/`、`Skills/` 是否存在，缺失则创建
2. 对缺失的记忆文件写入标准表头，不覆盖已有用户内容
3. 若 `session-handoff.md` 有未完成任务，则提示恢复入口
4. 返回“已加载/已创建/待确认”的初始化结果摘要

验收标准：
- 关键记忆文件全部存在且可读
- 用户知道当前是否存在旧会话与下一步建议

## 会话结束
用户说“先暂停”“下次继续”“先到这里”“结束会话”或输入 `/结束` 时：
1. 由 Coordinator 原生执行统一的结束会话流程
2. 若语义偏向“先暂停/下次继续”，走**轻量结束会话**：优先更新 `session-handoff.md`，写明当前状态为“待继续”或同等语义，并记录恢复入口与下一步
3. 若语义偏向“结束会话”或用户明确输入 `/结束`，走**标准结束会话**：生成完整交接并沉淀长期记忆（`project_scope.md`、`user_scope.md`、`knowledge/decisions.md`）
4. 如有未完成任务，统一写明恢复入口和下一步建议
5. 如本轮存在错误修复或模式沉淀，补写 `knowledge/errors.md`、`knowledge/patterns.md`、`Evolve/`

验收标准：
- `session-handoff.md` 必须包含“已完成 / 未完成 / 恢复入口 / 下一步”
- 轻量结束会话至少保证 `session-handoff.md` 完整可恢复；标准结束会话至少一个长期记忆文件完成更新，除非确实没有新增信息
- 回复中必须明确如何继续，而不是只说“好的，已结束”

## 框架重置

用户输入 `/重置` 时：
1. 先说明会被清理的内容，并要求用户二次确认
2. 将 `Core/记忆/` 备份到 `Backup/reset-时间戳/`
3. 清空并重建 `user_scope.md`、`project_scope.md`、`session-handoff.md` 和 `knowledge/` 模板
4. 保留 `.trae/`、`README.md`、`docs/`、`Builds/`、`Skills/`、`Backup/` 不变
5. 完成后提示用户可重新执行 `/初始化`
6. 若重置过程中失败，优先从最新备份回滚并记录失败原因

验收标准：
- 备份目录真实存在
- 新模板文件全部可读
- 回复中必须明确“已保留/已清理/如何恢复”

## 会话恢复
新会话检测到 session-handoff.md 时：
- 告知用户上次未完成工作
- 若检测到状态为“待继续”或同等轻量结束会话语义，优先提示用户可直接继续上次断点
- 用户确认→恢复任务状态继续执行
- 用户拒绝→询问是否开始新任务
