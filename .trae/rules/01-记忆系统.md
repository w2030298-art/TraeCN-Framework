# 记忆系统

## 记忆文件结构
```
Core/记忆/
├── user_scope.md        # 用户画像（长期）
├── project_scope.md     # 项目状态（中期）
├── session-handoff.md   # 会话交接（短期）
├── knowledge/           # 知识库
│   ├── errors.md        # 错误处理记录
│   ├── patterns.md      # 设计模式积累
│   └── decisions.md     # 关键决策记录
└── Evolve/              # 进化记录（跨项目/长期成果）
    ├── 进化日志.md      # 架构演进与能力提升
    └── 错题本.md        # 深度踩坑与预防措施
└── Modules/             # 复用资产库（跨项目/原子化代码）
    ├── Utils/           # 工具函数
    └── Components/      # 通用组件
```

## 加载策略（新会话自动执行）
1. 读取 `user_scope.md` → 了解用户偏好
2. 读取 `project_scope.md` → 了解项目状态
3. 检查 `session-handoff.md` → 存在则询问是否继续
4. 告知用户"记忆已加载"

## 更新策略
| 触发时机 | 更新文件 |
|----------|----------|
| 任务完成 | project_scope.md |
| 错误解决 | knowledge/errors.md |
| 关键决策 | knowledge/decisions.md |
| 会话结束 | session-handoff.md + 长期记忆沉淀 |
| 执行 `/重置` | 先备份 Core/记忆，再重建记忆模板 |
| 执行 `/复盘` | 更新 Evolve/ 下的所有文件 |
| 发现偏好 | user_scope.md |

### 会话结束时的长期记忆沉淀
当会话结束并执行交接时，自动沉淀以下信息：
- **project_scope.md**：添加本次会话的关键成果和版本更新
- **user_scope.md**：记录用户偏好和工作习惯
- **knowledge/decisions.md**：记录关键决策和背景理由

## 维护规则
- 同类错误只保留最新 5 条
- 单次写入不超过 50 行
- 会话交接继续后清空

## 跨项目进化 (Cross-project Evolution)
- **Evolve/** 目录下的内容是“只增不减”的（除非架构大重构）。
- 每次 `/复盘` 时，主 Agent 必须将本次会话中最具普适性的教训提炼并追加到 `Evolve/错题本.md`。
- 新项目初始化时，主 Agent 应优先读取 `Evolve/` 目录以避免重复历史错误。
