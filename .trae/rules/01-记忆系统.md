# 记忆系统

## 记忆文件结构
```
Core/记忆/
├── user_scope.md        # 用户画像（长期）
├── project_scope.md     # 项目状态（中期）
├── session-handoff.md   # 会话交接（短期）
└── knowledge/           # 知识库
    ├── errors.md        # 错误处理记录
    ├── patterns.md      # 设计模式积累
    └── decisions.md     # 关键决策记录
```

## 加载策略（新会话自动执行）
1. 读取 `user_scope.md` → 了解用户偏好
2. 读取 `project_scope.md` → 了解项目状态
3. 检查 `session-handoff.md` → 存在则询问是否继续
4. 告知用户"记忆已加载"

## 更新策略
| 触发时机 | 更新文件 |
|----------|----------|
| 任务完成 | project_scope.md |
| 错误解决 | knowledge/errors.md |
| 关键决策 | knowledge/decisions.md |
| 会话结束 | session-handoff.md |
| 发现偏好 | user_scope.md |

## 维护规则
- 同类错误只保留最新 5 条
- 单次写入不超过 50 行
- 会话交接继续后清空
